# Agent Guidelines

## Agent skills

This repo is configured for the Matt Pocock-inspired Codex skills:

- Use `$write-a-prd` to draft product requirements before larger feature work.
- Use `$prd-to-issues` to split approved PRDs into independently grabbable GitHub Issues.
- Use `$tdd` for test-first implementation of features and bug fixes.
- Use `$improve-codebase-architecture` for architecture review and refactoring opportunities.

Repo-specific configuration lives in `docs/agents/`:

- `docs/agents/issue-tracker.md` defines the issue tracker workflow.
- `docs/agents/triage-labels.md` defines labels and issue readiness states.
- `docs/agents/domain.md` defines the product and technical domain context.
