# Issue Tracker

This repo uses GitHub Issues for product planning, implementation tasks, bugs, and refactoring work.

## Repository

- GitHub repository: `jamiepratt/evolutionary-psychology-podcast`
- Remote: `git@github.com:jamiepratt/evolutionary-psychology-podcast.git`

## Workflow

Use GitHub Issues as the source of truth for planned work once a PRD or implementation plan has been approved.

When creating issues:

- Keep each issue independently grabbable.
- Include user-visible behavior, implementation notes, and validation expectations.
- Link back to the source PRD or planning document when one exists.
- Apply the appropriate type and status labels from `docs/agents/triage-labels.md`.

When working issues:

- Prefer small vertical slices that can be reviewed independently.
- Keep generated artifacts and source changes in the same issue when they must ship together.
- Mark blocked work clearly with the reason and dependency.
