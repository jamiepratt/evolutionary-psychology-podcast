# Triage Labels

Use these GitHub labels for PRDs, generated issues, and implementation work.

## Type Labels

- `type: prd`: Product requirements, specifications, and planning documents.
- `type: task`: Implementation work that adds or changes behavior.
- `type: bug`: Defects, regressions, or broken expected behavior.
- `type: refactor`: Architecture, maintainability, or internal cleanup work.

## Status Labels

- `status: needs triage`: Newly-created or unreviewed work that needs prioritization or clarification.
- `status: ready`: Work that is sufficiently specified and independently grabbable.
- `status: blocked`: Work that cannot proceed until a dependency, decision, or external condition is resolved.

## Readiness

An issue is `status: ready` when it has:

- A clear user or maintainer outcome.
- A bounded scope.
- Observable validation steps.
- Any dependencies or generated artifacts called out explicitly.
